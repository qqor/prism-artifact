[2;36m                    [0m         [39m   • Is configurable via standard Log4j2 property loading [0m[1;39m([0m[39m`PropertiesUtil`[0m[1;39m)[0m[39m.  [0m     [2m                [0m
[2;36m                    [0m         [39m   • Retains programmatic control via the existing `setEnableJndi`[0m[35m/[0m[39m`isEnableJndi` [0m  [2m                [0m
[2;36m                    [0m         [39mAPI for backward compatibility.[0m                                                     [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m2[0m[39m. Detailed steps  [0m                                                                 [2m                [0m
[2;36m                    [0m         [39m   a. Add a new configuration hook in `JndiLookup.java`:  [0m                          [2m                [0m
[2;36m                    [0m         [39m      – Import `org.apache.logging.log4j.util.PropertiesUtil`.  [0m                    [2m                [0m
[2;36m                    [0m         [39m      – In a static initializer, read the boolean system property [0m                  [2m                [0m
[2;36m                    [0m         [39m`log4j2.enableJndi` [0m[1;39m([0m[39mdefaulting to true for backward compatibility[0m[1;39m)[0m[39m and assign it [0m  [2m                [0m
[2;36m                    [0m         [39mto `ENABLE_JNDI`.  [0m                                                                 [2m                [0m
[2;36m                    [0m         [39m   b. Define a small set of known “remote” JNDI schemes [0m[1;39m([0m[39mldap, ldaps, rmi, dns, [0m    [2m                [0m
[2;36m                    [0m         [39miiop[0m[1;39m)[0m[39m.  [0m                                                                            [2m                [0m
[2;36m                    [0m         [39m   c. In the `[0m[1;35mlookup[0m[1;39m([0m[39mString key[0m[1;39m)[0m[39m` method [0m[1;39m([0m[39mthe main entry point[0m[1;39m)[0m[39m:  [0m                  [2m                [0m
[2;36m                    [0m         [39m      [0m[1;36m1[0m[39m. First, check `ENABLE_JNDI`. If false, log a status‐level warning and [0m      [2m                [0m
[2;36m                    [0m         [39mreturn null immediately.  [0m                                                          [2m                [0m
[2;36m                    [0m         [39m      [0m[1;36m2[0m[39m. Trim the incoming key and detect whether it is a container lookup [0m         [2m                [0m
[2;36m                    [0m         [1;39m([0m[39m`jav[0m[1;92ma:c[0m[39momp/env/…`[0m[1;39m)[0m[39m or a remote lookup:  [0m                                           [2m                [0m
[2;36m                    [0m         [39m         – If it starts with the container prefix, allow it to proceed.  [0m           [2m                [0m
[2;36m                    [0m         [39m         – Otherwise, inspect the substring before the first colon. If it matches [0m  [2m                [0m
[2;36m                    [0m         [39many of the remote schemes, log a warning and return null.  [0m                         [2m                [0m
[2;36m                    [0m         [39m      [0m[1;36m3[0m[39m. Delegate safe keys to the existing `InitialContext.lookup` logic [0m          [2m                [0m
[2;36m                    [0m         [39munmodified.  [0m                                                                       [2m                [0m
[2;36m                    [0m         [39m   d. Keep the existing setter/getter [0m[1;39m([0m[39m`setEnableJndi`, `isEnableJndi`[0m[1;39m)[0m[39m but [0m        [2m                [0m
[2;36m                    [0m         [39mdocument that programmatic calls after the static init will override the property [0m  [2m                [0m
[2;36m                    [0m         [39mvalue.[0m                                                                              [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m3[0m[39m. Why this is optimal  [0m                                                            [2m                [0m
[2;36m                    [0m         [39m   – It blocks the RCE‐enabled code path [0m[1;39m([0m[39mremote JNDI[0m[1;39m)[0m[39m at the earliest possible [0m    [2m                [0m
[2;36m                    [0m         [39mpoint.  [0m                                                                            [2m                [0m
[2;36m                    [0m         [39m   – It does not remove JNDI support entirely—container‐local lookups remain [0m       [2m                [0m

[2;36m                    [0m         [39m   – This approach mirrors upstream Log4j security patches [0m[1;39m([0m[39msee Log4j issue [0m        [2m                [0m
[2;36m                    [0m         [39mLOG4J2-[0m[1;36m3219[0m[39m/LOG4J2-[0m[1;36m3271[0m[1;39m)[0m[39m and uses the existing `PropertiesUtil` for consistency.  [0m  [2m                [0m
[2;36m                    [0m         [39m   – Fuzzing harness and test suites remain unchanged and continue to exercise the [0m [2m                [0m

[2;36m                    [0m         [39mtriggering a sanitizer exception or RCE.[0m                                            [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m4[0m[39m. Preserving critical logic  [0m                                                      [2m                [0m
[2;36m                    [0m         [39m   – The core `InitialContext.lookup` invocation and handling of local JNDI names [0m  [2m                [0m

[2;36m                    [0m         [39m   – No new external dependencies are introduced.  [0m                                 [2m                [0m
[2;36m                    [0m         [39m   – No changes to interpolation, substitution, or harness code.[0m                    [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
