[2;36m                    [0m         [1;36m1[0m[39m. Overview and Root Cause  [0m                                                        [2m                [0m
[2;36m                    [0m         [39m   The buffer overflow in aim_get_buddyname is caused by three mistakes:  [0m          [2m                [0m
[2;36m                    [0m         [39m   a[0m[1;39m)[0m[39m Copying from the wrong source offset [0m[1;39m([0m[39m`[0m[1;35mtvb_get_ptr[0m[1;39m([0m[39mtvb, offset, [0m[1;36m1[0m[1;39m)[0m[39m` instead [0m  [2m                [0m
[2;36m                    [0m         [39mof skipping the two‐byte length header[0m[1;39m)[0m[39m.  [0m                                          [2m                [0m
[2;36m                    [0m         [39m   b[0m[1;39m)[0m[39m No check that the [0m[1;36m16[0m[39m-bit length field fits into the fixed [0m[1;36m256[0m[39m-byte [0m           [2m                [0m
[2;36m                    [0m         [39mallocation.  [0m                                                                       [2m                [0m
[2;36m                    [0m         [39m   c[0m[1;39m)[0m[39m No null terminator added, nor any bounds enforcement.  [0m                       [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m2[0m[39m. Key Logic to Preserve  [0m                                                          [2m                [0m
[2;36m                    [0m         [39m   - Protocol semantics: the buddyname length is a [0m[1;36m16[0m[39m-bit big-endian word at the [0m   [2m                [0m
[2;36m                    [0m         [39mgiven offset.  [0m                                                                     [2m                [0m
[2;36m                    [0m         [39m   - Subsequent code expects aim_get_buddyname to return exactly the number of name[0m [2m                [0m
[2;36m                    [0m         [39mbytes [0m[1;39m([0m[39mexcluding the length header[0m[1;39m)[0m[39m.  [0m                                              [2m                [0m
[2;36m                    [0m         [39m   - The message parsing in dissect_aim_chat_outgoing_msg relies on “[0m[1;36m40[0m[39m + [0m          [2m                [0m
[2;36m                    [0m         [39mbuddyname_length” as the start of the chat payload; we will preserve that.  [0m        [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m3[0m[39m. Detailed Patch Steps in epan/dissectors/packet-aim.c  [0m                           [2m                [0m

[2;36m                    [0m         [39m      [0m[1;36m2[0m[39m. Compute the data start: `data_offset = offset + [0m[1;36m2[0m[39m;`  [0m                      [2m                [0m
[2;36m                    [0m         [39m      [0m[1;36m3[0m[39m. Clamp `buddyname_length` to what actually remains in the tvb:  [0m            [2m                [0m
[2;36m                    [0m         [39m         ```[0m                                                                        [2m                [0m
[2;36m                    [0m         [39m         reported = [0m[1;35mtvb_reported_length[0m[1;39m([0m[39mtvb[0m[1;39m)[0m[39m - data_offset;[0m                         [2m                [0m

[2;36m                    [0m         [39m             buddyname_length = reported;[0m                                           [2m                [0m
[2;36m                    [0m         [39m         ```  [0m                                                                      [2m                [0m
[2;36m                    [0m         [39m      [0m[1;36m4[0m[39m. Allocate exactly `buddyname_length + [0m[1;36m1[0m[39m` bytes from `pool` [0m[1;39m([0m[39mso no fixed [0m[1;36m256[0m [2m                [0m

[2;36m                    [0m         [39m      [0m[1;36m7[0m[39m. Return the adjusted `buddyname_length`.  [0m                                  [2m                [0m

[2;36m                    [0m         [1;36m4[0m[39m. Why This Fix Works  [0m                                                             [2m                [0m
[2;36m                    [0m         [39m   - Misalignment Fixed: by advancing the source pointer by [0m[1;36m2[0m[39m bytes, we copy only [0m  [2m                [0m
