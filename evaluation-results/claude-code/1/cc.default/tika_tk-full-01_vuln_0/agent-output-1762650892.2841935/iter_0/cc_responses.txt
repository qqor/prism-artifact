I've successfully patched the **Server Side Request Forgery (SSRF)** vulnerability in the XLIFF12Parser.

### The Problem
The XLIFF12Parser was vulnerable to XXE (XML External Entity) attacks, which allowed malicious XML input to cause the parser to make network connections to external URLs. This is what triggered the SSRF security issue when the fuzzer detected an attempted connection to `jazze.com:80`.

### The Fix

1. **Disabled external general entities** - Prevents loading external entities from URLs
2. **Disabled external parameter entities** - Prevents loading parameter entities from URLs  
3. **Disallowed DOCTYPE declarations** - Prevents DOCTYPE-based attacks
4. **Disabled external DTD loading** - Prevents loading external DTD files
5. **Disabled XInclude** - Prevents XInclude-based attacks

These security features prevent the XML parser from making any external network connections, effectively blocking SSRF attacks while still allowing the parser to function normally for legitimate XLIFF files.
