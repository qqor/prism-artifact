## SSRF Vulnerability Fix for XLIFF12Parser

**Problem:** The `XLIFF12Parser` was vulnerable to Server Side Request Forgery (SSRF) via XML External Entity (XXE) injection. The parser was creating a `SAXParserFactory` without disabling external entity resolution, allowing malicious XML files to make outbound network requests to arbitrary hosts.

**Root Cause:** The `_parse()` method was using `SAXParserFactory.newInstance()` directly and only setting `setNamespaceAware(true)`, without configuring security features to disable external entities.

**Fix Applied:**
1. Removed the unused `javax.xml.parsers.SAXParserFactory` import
2. Added import for `org.apache.tika.utils.XMLReaderUtils`
3. Changed the `_parse()` method to use `XMLReaderUtils.getSAXParserFactory()` instead of `SAXParserFactory.newInstance()`

The `XMLReaderUtils.getSAXParserFactory()` method returns a properly configured factory that:
- Sets `XMLConstants.FEATURE_SECURE_PROCESSING` to `true`
- Disables `external-general-entities`
- Disables `external-parameter-entities`
- Disables `load-external-dtd`
- Disables `load-dtd-grammar`

This prevents the XML parser from resolving external entities, which blocks the SSRF attack vector.
