## Fix Applied

**File:** `/home/user/crete/packages/python_oss_fuzz/.oss_fuzz/projects/pdfbox/project-parent/fuzz-targets/src/main/java/com/example/PDFExtractTextFuzzer.java`

**Problem:** The `PDDocument` was not being properly closed after use. The try-with-resources statement only managed the `InputStream`, but the `PDDocument` (which implements `Closeable`) was left open, causing resource leaks that led to the 25-second timeout.

**Fix:** Added `PDDocument` to the try-with-resources statement so it is automatically closed when the block exits:

```java
// Before (buggy):
try (InputStream is = new ByteArrayInputStream(bytes)) {
    PDDocument pdDocument = Loader.loadPDF(new RandomAccessReadBuffer(is));
    String txt = new PDFTextStripper().getText(pdDocument);
}

// After (fixed):
try (InputStream is = new ByteArrayInputStream(bytes);
     PDDocument pdDocument = Loader.loadPDF(new RandomAccessReadBuffer(is))) {
    String txt = new PDFTextStripper().getText(pdDocument);
}
```

This ensures that `PDDocument.close()` is called automatically when exiting the try block (either normally or due to an exception), properly releasing all resources and preventing the timeout.
