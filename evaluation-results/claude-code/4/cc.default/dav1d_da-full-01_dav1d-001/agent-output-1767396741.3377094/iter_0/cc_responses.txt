## Bug Analysis

The crash log showed:
```
bash: /out/dav1d_fuzzer_mt: Permission denied
```

**Root Cause:** The wrapper script `tests/libfuzzer/dav1d_fuzzer_mt@NO_OOM` was:
1. Setting execute permission on the binary before running it (line 17: `chmod +x`)
2. Running the binary
3. **Removing execute permission after running** (line 24: `chmod -x`)

This caused the binary to lose execute permissions after each run, making subsequent direct executions of `dav1d_fuzzer_mt` fail with "Permission denied".

## Fix Applied

Removed the line `chmod -x $this_dir/dav1d_fuzzer_mt` (and its associated comment) from the wrapper script. The binary now retains its execute permissions after the wrapper finishes, allowing subsequent runs to succeed.
