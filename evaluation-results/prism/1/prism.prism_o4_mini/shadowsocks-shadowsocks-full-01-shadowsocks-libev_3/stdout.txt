                             ## Fix Strategy                                                                                                                                    
                             <fix_strategy>                                                                                                                                     
                                                                                                                                                                                
                             1. Root Cause Analysis                                                                                                                             
                                - The core overflow happens because the parsing loop in `json_parse_ex` (around line 634 of src/shadowsocks/src/json.c) uses                    
                                     json_char b = (state.ptr == end ? 0 : *state.ptr);                                                                                         
                                  but increments `state.ptr` unconditionally in the `for` header (`++state.ptr`). When `state.ptr` advances from `end` to                       
                             `end+1`, the condition `state.ptr == end` is false, so `*state.ptr` dereferences one byte past the allocated buffer, triggering a                  
                             heap-buffer-overflow.                                                                                                                              
                                                                                                                                                                                
                             2. Preservation of Existing Logic                                                                                                                  
                                - All existing logic relies on treating a zero-valued `b` as EOF or string-terminator, triggering either a `break` (in                          
                             non-string contexts) or an error (in string contexts). We must preserve that exact behavior for `state.ptr == end`.                                
                                - We must not alter any fuzzing harness, public interface, or memory-management utility beyond this single loop.                                
                                                                                                                                                                                
                             3. Patch Outline
