                             ## Fix Strategy                                                                                                                                                                     
                             <fix_strategy>                                                                                                                                                                      
                                                                                                                                                                                                                 
                             1. Introduce per-segment and overall limits in PfbParser to cap PFB headerdeclared sizes:                                                                                          
                                a. In PfbParser (parsePfb), define a constant                                                                                                                                    
                                   `private static final long MAX_PFB_SEGMENT_LENGTH = 1 * 1024 * 1024; // 1 MB`                                                                                                 
                                b. Immediately after reading the 4-byte `size` from the header, add:                                                                                                             
                                   ```java                                                                                                                                                                       
                                   if (size < 0 || size > MAX_PFB_SEGMENT_LENGTH) {                                                                                                                              
                                       throw new IOException("PFB segment size out of bounds: " + size);                                                                                                         
                                   }                                                                                                                                                                             
                                   ```                                                                                                                                                                           
                                This prevents any single ASCII or binary segment from exceeding 1 MB, bounding subsequent decryption and lexing work.                                                            
                                                                                                                                                                                                                 
                             2. Bound decrypted data passed to the Type1 lexer:                                                                                                                                  
                                a. In Type1Parser.parseBinary, after computing `byte[] decrypted`, introduce                                                                                                     
                                   ```java                                                                                                                                                                       
                                   private static final int MAX_DECRYPTED_LENGTH = 1 * 1024 * 1024; // 1 MB                                                                                                      
                                   if (decrypted.length > MAX_DECRYPTED_LENGTH) {                                                                                                                                
                                       throw new IOException("Decrypted font data too large: " + decrypted.length);                                                                                              
                                   }                                                                                                                                                                             
                                   ```                                                                                                                                                                           
                                This aborts parsing when a declared PFB segment is nominally valid but decrypts to a huge payload.                                                                               
                                                                                                                                                                                                                 
                             3. Prevent unbounded tokenscanning in Type1Lexer.readToken:                                                                                                                        
                                a. In Type1Lexer, define                                                                                                                                                         
                                   `private static final int MAX_TOKEN_SCAN_BYTES = 16 * 1024; // e.g. 16 KB per token`                                                                                          
                                b. At the start of `readToken(Token prevToken)`, initialize a local counter, e.g.                                                                                                
                                   `int scanned = 0;`                                                                                                                                                            
                                c. On each `getChar()` or buffer position advance, increment `scanned` by 1; if `scanned > MAX_TOKEN_SCAN_BYTES`, throw                                                          
                                   `throw new DamagedFontException("Token exceeds max length at position " + buffer.position());`                                                                                
                                This ensures that a malformed font stream cannot loop indefinitely looking for delimiters or token patterns.                                                                     
                                                                                                                                                                                                                 
                             4. Clamp loops driven by integer counts in Type1Parser:                                                                                                                             
                                a. Define safe maxima in Type1Parser, e.g.                                                                                                                                       
                                   ```java                                                                                                                                                                       
                                   private static final int MAX_CHARSTRINGS = 1_000;                                                                                                                             
                                   private static final int MAX_SUBRS      = 5_000;                                                                                                                              
                                   private static final int MAX_OTHERSUBRS = 5_000;                                                                                                                              
                                   ```                                                                                                                                                                           
                                b. In `readCharStrings(int lenIV)`, after                                                                                                                                        
                                   `int length = read(Token.INTEGER).intValue();`                                                                                                                                
                                   add:                                                                                                                                                                          
                                   ```java                                                                                                                                                                       
                                   if (length < 0) {                                                                                                                                                             
                                       throw new IOException("Negative CharStrings count: " + length);                                                                                                           
                                   }                                                                                                                                                                             
                                   if (length > MAX_CHARSTRINGS) {                                                                                                                                               
                                       LOG.warn("Clamping CharStrings count {} to {}", length, MAX_CHARSTRINGS);                                                                                                 
                                       length = MAX_CHARSTRINGS;                                                                                                                                                 
                                   }                                                                                                                                                                             
                                   ```                                                                                                                                                                           
                                c. Similarly in `readSubrs(int lenIV)` and `readOtherSubrs()`, clamp their `length` values to `MAX_SUBRS` and `MAX_OTHERSUBRS`, respectively, logging a warning                  
                             and either throwing or reducing the loop bound.                                                                                                                                     
                                                                                                                                                                                                                 
                             5. Ensure these exceptions propagate out of font parsing and abort parseBinary, causing PDFTextStripper.getText to throw IOException rather than hang:                              
                                - The existing code already allows IOExceptions (and DamagedFontException) to bubble up, aborting text extraction.                                                               
                                - No changes to harness or fuzzing logic are needed.                                                                                                                             
                                                                                                                                                                                                                 
                             Justification:                                                                                                                                                                      
                             - By enforcing strict persegment and decryption length limits, we eliminate the root cause: malicious or malformed PFBs that declare huge segments leading to                      
                             prolonged decryption and lexing.                                                                                                                                                    
                             - Tokenlevel scanning is now bounded, guaranteeing progress and avoiding infinite loops in readToken.                                                                              
                             - Clamping CharStrings, Subrs, and OtherSubrs counts stops excessive parser iterations driven by large integer fields.                                                              
                             - Throwing IOException or DamagedFontException on overlimit inputs preserves legitimate parsing for wellformed, reasonable fonts, while exposing errors to the                    
                             fuzzer as recoverable exceptions, avoiding hangs.                                                                                                                                   
                             - Existing functionality is preserved for valid fonts; only extreme or malformed cases are rejected early.
