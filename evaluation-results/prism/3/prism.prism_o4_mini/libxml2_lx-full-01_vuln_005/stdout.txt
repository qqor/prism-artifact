                             ## Summary

                             within `api.c` when handling a comment node. A memory region allocated by `xmlStrdup` (via `xmlFuzzMalloc`) was freed twice during a single input execution,                        
                             leading to a double-free error and abort.                                                                                                                                           
                                                                                                                                                                                                                 
                             ## Description                                                                                                                                                                      
                                                                                                                                                                                                                 
                              Issue type                                                                                                                                                                        
                                Memory corruption: double-free vulnerability.                                                                                                                                   
                                                                                                                                                                                                                 
                              Fault details                                                                                                                                                                     
                                The sanitizer report shows an ERROR: AddressSanitizer: attempting double-free on a small (11-byte) heap buffer at address `0x5020000000b0`.                                   
                                The buffer was originally allocated in `xmlStrdup` (called from `xmlNewComment` in `tree.c`).                                                                                   
                                The first free occurs inside `LLVMFuzzerTestOneInput` at line 3580 of `api.c`. A second free of the same address is triggered moments later in the same fuzz
