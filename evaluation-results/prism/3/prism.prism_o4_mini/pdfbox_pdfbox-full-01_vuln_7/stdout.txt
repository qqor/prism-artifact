                               During fuzzing, the `fuzzerTestOneInput` method in `PDFExtractTextFuzzer` invokes Apache PDFBoxs `PDFTextStripper.getText`. That call traverses the PDF page                     
                             tree (`PDPageTree`) and eventually triggers the JVMs dynamic bytecode generation path (ASMs `AnnotationWriter`) while trying to instrument or link method                         
                             handles. The main thread remains in that compute-intensive loop until the fuzzer kills it for timeout.                                                                              
                                                                                                                                                                                                                 
                             - Important observations                                                                                                                                                            
                                The thread dump shows the main thread in state RUNNABLE, repeatedly in `AnnotationWriter.computeAnnotationsSize`  `MethodWriter.computeMethodInfoSize`                      
                             `ClassWriter.toByteArray`  `InvokerBytecodeGenerator`  `LambdaForm.compileToBytecode`.                                                                                            
                                All other application threads are either idle or performing unrelated GC/ref-cleaning work.                                                                                     
                                There is no JVM crash, exception stack trace, or memory erroronly a lack of termination.                                                                                       
                                The code path originates in `PDFExtractTextFuzzer.fuzzerTestOneInput` (line 41), goes through PDFBoxs page tree iterator (`PDPageTree.iterator`) and text                      
                             stripper (`PDFTextStripper.processPages`/`writeText`), and then into the JVMs invoke/link bytecode emitter.                                                                        
                                                                                                                                                                                                                 
                             - Areas to investigate                                                                                                                                                              
                               1. The PDFBox page-tree iteration logic (`PDPageTree.isPageTreeNode`, `enqueueKids`, `PageIterator`) for unusually deep or malformed page structures that might                   
                             lead to unbounded work.                                                                                                                                                             
                               2. Interaction between PDFBoxs model (annotations, page dictionaries) and the JVM instrumentation/drivers (Jazzer) invoking ASM to generate or transform lambda                  
                             forms and method handles.                                                                                                                                                           
                               3. The harness entry point (`PDFExtractTextFuzzer.fuzzerTestOneInput`) to confirm how input bytes map to PDF parsing and whether certain malformed PDFs trigger

                             ## Fix Strategy                                                                                                                                                                     
                             <fix_strategy>                                                                                                                                                                      
                                                                                                                                                                                                                 
                             1. Root Cause and Symptom                                                                                                                                                           
                                - The hang arises in PDPageTree.PageIterator, whose constructor loops over page-tree nodes via `enqueueKids` without any cycle detection. Malformed PDFs with                    
                             cyclic Kids references cause `kids` never to empty out, leading to an infinite loop and the observed timeout.                                                                     
                                - Other code paths in PDFBox (e.g., the depth-first `get(int,)` and PDFieldTree traversal) already employ a `Set` to track visited COSDictionary objects and                    
                             break on recursion. PageIterator is missing this proven pattern.                                                                                                                    
                                                                                                                                                                                                                 
                             2. High-Level Patch Plan                                                                                                                                                            
                                 Introduce a `Set<COSDictionary>` (e.g. `visitedNodes`) in the inner `PageIterator` class to record all node dictionaries seen so far.                                          
                                 At the start of processing each node in `enqueueKids`, check if it's already in `visitedNodes`. If yes, log a warning and return an empty list to omit further                 
                             processing.                                                                                                                                                                         
                                 Otherwise, add it to `visitedNodes`, then proceed as before:                                                                                                                   
                                   If `isPageTreeNode(node)`, call `getKids(node)` to collect child nodes.                                                                                                      
                                   If its an actual page (`TYPE` == PAGE), add it to `queue`.                                                                                                                  
                                 This change prevents `kids` from ever containing a previously seen node, ensuring the whileloop in the constructor will terminate even on cyclic structures.                  
                                                                                                                                                                                                                 
                             3. Detailed Implementation Steps                                                                                                                                                    
                                a. In PDPageTree.java, locate the `private final class PageIterator implements Iterator<PDPage>` (around lines 180197).                                                         
                                b. Add a field:                                                                                                                                                                  
                                   ```                                                                                                                                                                           
                                   private final Set<COSDictionary> visitedNodes = new HashSet<>();                                                                                                              
                                   ```                                                                                                                                                                           
                                c. In the `PageIterator(COSDictionary node)` constructor, initialize the root:                                                                                                   
                                   ```                                                                                                                                                                           
                                   if (node != null) {                                                                                                                                                           
                                       visitedNodes.add(node);                                                                                                                                                   
                                   }                                                                                                                                                                             
                                   ```                                                                                                                                                                           
                                d. Replace the current `enqueueKids(COSDictionary node)` implementation (lines ~205225) with:                                                                                   
                                   ```                                                                                                                                                                           
                                   private List<COSDictionary> enqueueKids(COSDictionary node) {                                                                                                                 
                                       if (node == null) {                                                                                                                                                       
                                           return Collections.emptyList();                                                                                                                                       
                                       }                                                                                                                                                                         
                                       // Cycle detection: skip already visited nodes                                                                                                                            
                                       if (visitedNodes.contains(node)) {                                                                                                                                        
                                           LOG.error("Skipping cyclic page-tree node {}", node);                                                                                                                 
                                           return Collections.emptyList();                                                                                                                                       
                                       }                                                                                                                                                                         
                                       visitedNodes.add(node);                                                                                                                                                   
                                       // Internal nodes: enqueue descendants                                                                                                                                    
                                       if (isPageTreeNode(node)) {                                                                                                                                               
                                           List<COSDictionary> kids = getKids(node);                                                                                                                             
                                           return kids;                                                                                                                                                          
                                       }                                                                                                                                                                         
                                       // Leaf pages: queue for iteration                                                                                                                                        
                                       COSName type = node.getCOSName(COSName.TYPE);                                                                                                                             
                                       if (COSName.PAGE.equals(type)) {                                                                                                                                          
                                           queue.add(node);                                                                                                                                                      
                                       } else {                                                                                                                                                                  
                                           LOG.error("Page skipped due to an invalid or missing type {}", () ->                                                                                                  
                                                     (type == null ? "(null)" : type));                                                                                                                          
                                       }                                                                                                                                                                         
                                       return Collections.emptyList();                                                                                                                                           
                                   }                                                                                                                                                                             
                                   ```                                                                                                                                                                           
                                e. Cleanup imports if needed (`java.util.HashSet`, `java.util.Set`).                                                                                                             
                                                                                                                                                                                                                 
                             4. Preservation of Existing Functionality                                                                                                                                           
                                - Well-formed PDFs (acyclic page trees) are unaffected: every node is visited exactly once, and the traversal order remains BFS-style.                                           
                                - Existing tests for PDFTextStripper and PDPageTree will still pass, as the visible API (`iterator()`, `getText()`) behavior is identical when no cycles exist.                  
                                - We simply guard against pathological inputs that previously caused non-termination under fuzzing.                                                                              
                                                                                                                                                                                                                 
                             5. Justification and Side-Effects                                                                                                                                                   
                                - Using a `Set` for cycle detection is a lightweight and proven technique, mirrored by other parts of the codebase (PDFieldTree, depth-first `get(...)`).                        
                                - It ensures the fuzzer harness (`PDFExtractTextFuzzer`) can terminate promptly on malicious or malformed inputs, eliminating the observed libFuzzer                             
                             timeout/hang without touching any harness or fuzzing logic.                                                                                                                         
                                - Potential side-effect: in PDFs with a legitimate deep tree, the memory overhead for `visitedNodes` is O(number of nodes) but remains bounded by typical                        
                             document sizes.                                                                                                                                                                     
                                - No exception throwing is introduced; we simply log and skip cycles, preserving best-effort extraction semantics.
