                             # Evaluation Report                                                                                                                                                                 
                                                                                                                                                                                                                 
                             ## Summary                                                                                                                                                                          
                                                                                                                                                                                                                 
                             A heap-buffer-overflow was detected by AddressSanitizer in the JSON parsing code of the shadowsocks project. The crash occurs in the function `json_parse_ex` (file                 
                             `json.c`, line 603), where a onebyte read is performed just past the end of a small heap allocation. The issue was uncovered by the `json_fuzz` fuzzing harness                    
                             under libFuzzer, indicating an outofbounds memory access when parsing certain inputs.                                                                                             
                                                                                                                                                                                                                 
                             ## Description                                                                                                                                                                      
                                                                                                                                                                                                                 
                             - Issue Type                                                                                                                                                                        
                               A onebyte read beyond the end of a 7byte heap buffer (heap-buffer-overflow), as reported by AddressSanitizer.                                                                   
                                                                                                                                                                                                                 
                             - Vulnerability Classification                                                                                                                                                      
                               Memory safety violation: heap-buffer-overflow.                                                                                                                                    
                                                                                                                                                                                                                 
                             - Observations & Findings                                                                                                                                                           
                                The crash stack trace shows the overflow originates in `json_parse_ex` at `json.c:603:55`.                                                                                      
                                The buffer in question spans addresses `[0x5020000000b0,0x5020000000b7)` and the read occurs at `0x5020000000b7`, exactly one byte past the allocation.                         
                                The faulty read propagates through `json_parse` (line 958) and into the fuzz entry point `LLVMFuzzerTestOneInput` in `json_fuzz.c`.                                             
                                AddressSanitizers shadow memory log confirms the adjacent byte is in the heap redzone, marking the overflow.                                                                   
                                The harness invoked is `out/json_fuzz -rss_limit_mb=2560 -timeout=25 -runs=100 /testcase`, demonstrating the issue under repeated fuzz iterations.                              
                                                                                                                                                                                                                 
                             - Code Areas for Investigation                                                                                                                                                      
                               1. **json_parse_ex (json.c around line 603):** inspect buffer accesses, bounds checks, and how input length is computed and enforced.                                             
                               2. **json_parse (json.c around line 958):** review higherlevel parsing logic that calls `json_parse_ex` and any intermediate data structures.                                    
                               3. **Fuzz harness (json_fuzz.c at LLVMFuzzerTestOneInput):** understand how test inputs are fed into the parser and whether there are assumptions about                           
                             termination or buffer size.                                                                                                                                                         
                               4. **Memory management patterns:** look at how parser buffers are allocated, reallocated, and freed, as well as any use of `malloc`/`new` surrounding the parsing                 
                             routines.                                                                                                                                                                           
                                                                                                                                                                                                                 
                             - Holistic View                                                                                                                                                                     
                               To fully understand the impact and context of this overflow, review the entire JSON parser module in `src/shadowsocks/src/json.c`, including any utility

                             ## Fix Strategy                                                                                                                                                                     
                             <fix_strategy>                                                                                                                                                                      
                                                                                                                                                                                                                 
                             1. Root Cause Analysis                                                                                                                                                              
                                - The overflow occurs in json_parse_ex when parsing the JSON literals true, false, and null. The code checks if there are enough remaining bytes using                     
                             `(end - state.ptr) < N`, but `N` underestimates the required look-ahead by one. As a result, for inputs whose length exactly matches the literal length, the parser                 
                             reads one byte past the buffer end.                                                                                                                                                 
                                                                                                                                                                                                                 
                             2. Targeted Patches                                                                                                                                                                 
                                - In `src/json.c`, within the `switch (b)` in `json_parse_ex`:                                                                                                                   
                                  a. **Case 't' (true)** at ~line 602:                                                                                                                                           
                                     - Original:                                                                                                                                                                 
                                       `if ((end - state.ptr) < 3 || *(++state.ptr) != 'r' || ... )`                                                                                                             
                                     - Change the check to `< 4` since true is 4 bytes.                                                                                                                        
                                  b. **Case 'f' (false)** at ~line 618:                                                                                                                                          
                                     - Original:                                                                                                                                                                 
                                       `if ((end - state.ptr) < 4 || *(++state.ptr) != 'a' || ... )`                                                                                                             
                                     - Change the check to `< 5` since false is 5 bytes.                                                                                                                       
                                  c. **Case 'n' (null)** at ~line 633:                                                                                                                                           
                                     - Original:                                                                                                                                                                 
                                       `if ((end - state.ptr) < 3 || *(++state.ptr) != 'u' || ... )`                                                                                                             
                                     - Change the check to `< 4` since null is 4 bytes.                                                                                                                        
                                                                                                                                                                                                                 
                             3. Implementation Steps                                                                                                                                                             
                                a. Open `src/json.c` and locate the literal parsing blocks in `json_parse_ex`.                                                                                                   
                                b. For each of the three cases, update the constant in the `(end - state.ptr) < N` condition as described.                                                                       
                                c. Ensure parentheses and spacing match the surrounding style.                                                                                                                   
                                d. Rebuild the project and rerun the fuzz harness (`json_fuzz`).
