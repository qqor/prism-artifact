[2;36m                    [0m         [1;36m1[0m[39m. **Root Cause Analysis**  [0m                                                        [2m                [0m
[2;36m                    [0m         [39m   The crash stems from this line in `apple/exif-mnote-data-apple.c`:[0m               [2m                [0m
[2;36m                    [0m         [39m     ```[0m                                                                            [2m                [0m
[2;36m                    [0m         [39m     if [0m[1;39m([0m[39m![0m[1;35mstrcmp[0m[1;39m([0m[1;39m([0m[39mconst char *[0m[1;39m)[0m[39m e->data, [0m[32m"Apple iOS"[0m[1;39m)[0m[1;39m)[0m[39m [0m[1;39m{[0m                            [2m                [0m
[2;36m                    [0m         [39m     ```[0m                                                                            [2m                [0m
[2;36m                    [0m         [39m   Here, `e->data` is a raw blob of length `e->size` [0m[1;39m([0m[39mas allocated by [0m              [2m                [0m
[2;36m                    [0m         [39m`exif_data_alloc`[0m[1;39m)[0m[39m, without any guaranteed NUL-terminator. When `e->size` is less [0m  [2m                [0m
[2;36m                    [0m         [39mthan `[0m[1;35mstrlen[0m[1;39m([0m[32m"Apple iOS"[0m[1;39m)[0m[39m+[0m[1;36m1[0m[39m`, `strcmp` overruns the buffer, reading past its end [0m   [2m                [0m
[2;36m                    [0m         [39mand triggering the AddressSanitizer heap-buffer-overflow.[0m                           [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m2[0m[39m. **Preserve Existing Behavior**  [0m                                                 [2m                [0m
[2;36m                    [0m         [39m   - We still need to detect whether the maker-note data begins with the ASCII [0m     [2m                [0m
[2;36m                    [0m         [39msequence `[0m[32m"Apple iOS"[0m[39m`.  [0m                                                           [2m                [0m
[2;36m                    [0m         [39m   - We must not alter any higher-level logic [0m[1;39m([0m[39me.g., how `interpret_maker_note` [0m    [2m                [0m
[2;36m                    [0m         [39mchooses handler variants based on this check[0m[1;39m)[0m[39m.[0m                                      [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m3[0m[39m. **Proposed Safe Comparison**  [0m                                                   [2m                [0m
[2;36m                    [0m         [39m   Mirror the pattern used in `exif-mnote-data-fuji.c`:  [0m                           [2m                [0m
[2;36m                    [0m         [39m   - First check that `e->size` is large enough to contain the prefix.  [0m            [2m                [0m
[2;36m                    [0m         [39m   - Then compare only the prefix bytes via `memcmp` [0m[1;39m([0m[39mor `strncmp`[0m[1;39m)[0m[39m so no reads [0m    [2m                [0m
[2;36m                    [0m         [39moccur past the buffer.  [0m                                                            [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m4[0m[39m. **Patch Steps**  [0m                                                                [2m                [0m
[2;36m                    [0m         [39m   a. Open `libexif/apple/exif-mnote-data-apple.c` and locate [0m                      [2m                [0m
[2;36m                    [0m         [39m`exif_mnote_data_apple_identify`, around line [0m[1;36m236[0m[39m.  [0m                                [2m                [0m
[2;36m                    [0m         [39m   b. Define a constant for the ASCII prefix and its length:[0m                        [2m                [0m
[2;36m                    [0m         [39m      ```c[0m                                                                          [2m                [0m
[2;36m                    [0m         [39m      [0m[35m/[0m[39m* At top of function or file scope *[0m[35m/[0m                                        [2m                [0m
[2;36m                    [0m         [39m      static const char APPLE_PREFIX[0m[1;39m[[0m[1;39m][0m[39m = [0m[32m"Apple iOS"[0m[39m;[0m                               [2m                [0m

[2;36m                    [0m         [39m      ```[0m                                                                           [2m                [0m
[2;36m                    [0m         [39m   c. Replace the unsafe `strcmp` call with a bounded check:[0m                        [2m                [0m
[2;36m                    [0m         [39m      ```diff[0m                                                                       [2m                [0m
[2;36m                    [0m         [39m      -    if [0m[1;39m([0m[39m![0m[1;35mstrcmp[0m[1;39m([0m[1;39m([0m[39mconst char *[0m[1;39m)[0m[39m e->data, [0m[32m"Apple iOS"[0m[1;39m)[0m[1;39m)[0m[39m [0m[1;39m{[0m                      [2m                [0m
[2;36m                    [0m         [39m      +    [0m[35m/[0m[39m* Only compare up to APPLE_PREFIX_LEN bytes, ensuring buffer is large [0m  [2m                [0m
[2;36m                    [0m         [39menough *[0m[35m/[0m                                                                           [2m                [0m
[2;36m                    [0m         [39m      +    if [0m[1;39m([0m[39me->size >= APPLE_PREFIX_LEN[0m                                          [2m                [0m
[2;36m                    [0m         [39m      +        && [0m[1;35mmemcmp[0m[1;39m([0m[39me->data, APPLE_PREFIX, APPLE_PREFIX_LEN[0m[1;39m)[0m[39m == [0m[1;36m0[0m[1;39m)[0m[39m [0m[1;39m{[0m           [2m                [0m
[2;36m                    [0m         [39m      ```[0m                                                                           [2m                [0m
[2;36m                    [0m         [39m   d. Leave the `variant = [0m[1;36m1[0m[39m;` [0m[35m/[0m[39m `variant = [0m[1;36m0[0m[39m;` logic unchanged, so callers behave [0m [2m                [0m
[2;36m                    [0m         [39midentically when the prefix matches.[0m                                                [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
[2;36m                    [0m         [1;36m5[0m[39m. **Why This Works**  [0m                                                             [2m                [0m
[2;36m                    [0m         [39m   - **No heap-overreads**: we check `e->size >= APPLE_PREFIX_LEN` before reading, [0m [2m                [0m
[2;36m                    [0m         [39mso `memcmp` never touches out-of-bounds memory.  [0m                                   [2m                [0m
[2;36m                    [0m         [39m   - **Retains semantics**: we still detect the Apple maker-note variant by [0m        [2m                [0m
[2;36m                    [0m         [39mmatching the exact ASCII signature.  [0m                                               [2m                [0m
[2;36m                    [0m         [39m   - **Minimal impact**: only one function is altered; all existing tests and fuzz [0m [2m                [0m

[2;36m                    [0m                                                                                             [2m                [0m

[2;36m                    [0m         [39many new risk.  [0m                                                                     [2m                [0m
[2;36m                    [0m                                                                                             [2m                [0m
